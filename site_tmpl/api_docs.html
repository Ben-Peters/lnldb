WPI Lens and Lights API

The LNL API is a simple REST API which can be used to connect apps and services with the LNL Database (LNLDB). The LNL API currently provides read-only access to information already available on our public-facing webpages.

Getting Started

The LNL API can be accessed through standard HTTP requests and will return JSON metadata. To access protected information, your application will be required to authenticate through the LNLDB first. The API is currently configured to respond to the following HTTP Methods:

Method Action
GET Retrieves the specified resources
HEAD Identical to GET request, except without the response body
OPTIONS Provides communication options for the endpoint
Note: If attempting to communicate with the API through methods such as ajax, note that by default, CORS prohibits cross-origin requests. You may need to register your application with the API before using such methods. Contact lnl-w@wpi.edu for assistance.

Authentication


Session-based

Some endpoints access data that requires authentication. To make the process as simple as possible, the API utilizes session-based authentication through the LNLDB. If the user does not have an active session, or their current session has expired, simply redirect them to the login page.

Note: Endpoints that require authentication are currently unavailable as we work to update authentication methods site-wide.
